## Spacing Scale

Declared values (multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline badge padding, `gap-1` |
| sm | 8px | Bubble internal padding (vertical), tool card gaps, `gap-2` |
| lg | 16px | Column padding, stream padding, `px-4` |
| xl | 24px | Section spacing |
| 2xl | 32px | Layout column widths modulo |
| 3xl | 48px | Major vertical section breaks |

Exceptions (component-specific sizes, not spacing scale tokens):
- `12px` (`px-3`): bubble padding horizontal, composer area padding, tool card header padding — component-specific size, not a spacing scale token
- Nav column fixed at 56px (not a standard scale value; locked by D-07a)
- Session list fixed at 280px (locked by D-07b)
- Touch targets: minimum 44×44px on mobile (accessibility requirement)
- Streaming cursor: 2×14px (exact visual spec from mockup)

## Interaction Contracts

### Composer / Input Area

| Property | Spec |
|----------|------|
| Component | `Textarea` from `@tether/design` |
| Default height | 36px (1 line) |
| Max height | 120px (auto-grow, then scroll internally) |
| Grow behavior | `scrollHeight` auto-size on `input` event; capped at 120px |
| Border | `border-border`, on focus `ring-2 ring-ring` (indigo via token override at component level: `ring-[#6366f1]`) |
| Background | `bg-card` (surface-2 equivalent) |
| Placeholder | zh: `"发送给 Claude…"` / en: `"Send to Claude…"` (model name dynamic) |
| Send button | Right of textarea, 36×36px, `bg-brand text-foreground-inverse`; icon: `Send` (lucide) |
| Send disabled | When textarea empty OR session in-flight: `opacity-50 cursor-not-allowed` |
| Send loading | In-flight: replace `Send` icon with `Loader2` spinning |
| Keyboard submit | `Enter` = send; `Shift+Enter` = newline |
| Mobile bottom | `padding-bottom: max(10px, env(safe-area-inset-bottom))` |

### Streaming State (in-flight, D-44)

| Element | In-flight behavior |
|---------|-------------------|
| Composer textarea | `disabled`, `opacity-50` |
| Send button | `disabled`, shows `Loader2` spinner |
| Agent bubble | Appears immediately with `StreamingCursor` after send |
| `agent.delta` | Accumulated text re-rendered through react-markdown in real-time |
| Unclosed code block | Rendered as raw text fallback during streaming; re-rendered fully on `agent.result` |
| `agent.result` | Cursor removed, `ResultCard` appended below bubble content, composer unlocked |
| Input after unlock | Focus returns to textarea automatically |

### "AI is responding..." Placeholder (D-45)

Shown when: HTTP history load finds last record is `user.message` with no `agent.result`.

| Property | Spec |
|----------|------|
| Visual | Agent bubble (`bg-agent-bubble border-border`) containing `ThinkingDots` component |
| ThinkingDots | 3 × 6px dots, `bg-foreground-tertiary`, bounce animation staggered 0/160ms/320ms |
| Session locked | Composer disabled, send disabled (same as in-flight) |
| On WS reconnect with catchup | Replace ThinkingDots with accumulated text + streaming cursor |
| On WS reconnect no catchup | Replace with "回复丢失" error state (see below) |

### "Reply Lost" Error State (D-45 edge case)

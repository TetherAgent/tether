**ChatSessionRunner class**：

```typescript
const VALID_PROVIDERS = new Set(['claude', 'codex', 'opencode']);

export class ChatSessionRunner implements IChatRunner {
  private readonly activeSubprocesses = new Map<string, ActiveSubprocess>();

  constructor(private readonly options: ChatRunnerOptions) {}

  /** 执行一条消息：新会话 or 续接 */
  async run(params: {
    clientId: string;
    sessionId: string | null;   // null = 首条消息，需隐式创建
    provider: string;
    model: string;
    cwd: string;
    message: string;
  } | {
    clientId: string;
    sessionId: string;          // 续接消息
    message: string;
  }): Promise<void>

  /** 客户端重连后获取 catchup 文本（若有活跃 subprocess） */
  getCatchup(sessionId: string): string | undefined

  /** 杀死某 session 的活跃 subprocess */
  kill(sessionId: string): void
}
```

**run() 实现要点**（逐步描述，executor 需完整实现）：

1. **参数判断**：
   - 如果 sessionId 为 null，需创建新 session（调用 `store.createSession(...)` 生成 UUID，transport='chat'，provider 必须在 VALID_PROVIDERS 中，否则 onError）
   - 创建成功后立即调用 `onSessionCreated(clientId, newSessionId)`
   - 立即写 `store.appendChatEvent(sessionId, 'user.message', { message })`

2. **查 agentSessionId**（per D-22）：
   - 从 `store.getSession(sessionId)` 取 `agentSessionId`
   - 若本地无则暂用 undefined（跨设备限制 D-35）

3. **spawn（per D-01b）**：
   ```typescript
   const args = [
     '-p', message,
     '--output-format', 'stream-json',
     ...(agentSessionId ? ['--resume', agentSessionId] : [])
   ];
   const child = spawn('claude', args, {
     cwd,
     stdio: ['ignore', 'pipe', 'pipe'],
     env: process.env
   });
   ```
   **注意**：`ChatSessionRunner` 仅实现 provider='claude' 的 spawn 逻辑（per D-01b）。`CodexChatRunner` 和 `CopilotChatRunner` 以 stub 形式单独在同文件实现（见上方 IChatRunner 接口说明），stub 的 run() 直接调用 onError('provider_not_supported', ...)，不执行 spawn。

4. **注册到 activeSubprocesses Map**：
   ```typescript
   this.activeSubprocesses.set(sessionId, {
     process: child, accumulatedText: '', startedAt: Date.now(), clientId
   });
   ```

5. **stdout 解析（NDJSON 逐行）**（per D-33 不写 delta，只写 agent.result）：
   ```typescript
   let lineBuffer = '';
   child.stdout.on('data', (chunk: Buffer) => {
     lineBuffer += chunk.toString();
     const lines = lineBuffer.split('\n');
     lineBuffer = lines.pop() ?? '';
     for (const line of lines) {
       if (!line.trim()) continue;
       try {
         const event = JSON.parse(line) as Record<string, unknown>;
         this.handleStreamEvent(event, sessionId, clientId);
       } catch { /* skip bad line */ }
     }
   });
   ```

6. **handleStreamEvent** 处理以下 event.type：
   - `'content_block_delta'`：当 `event.delta?.type === 'text_delta'`，取 `event.delta.text` 追加到 `activeSubprocesses.get(sessionId).accumulatedText`，调用 `onDelta`
   - `'message_stop'`：取 accumulatedText、usage（从 event 或前面的 message 事件中取），调用 `store.appendChatEvent(sessionId, 'agent.result', { text: accumulatedText, usage })`，若 event 中有 session_id 字段，调用 `store.updateSessionAgentId(sessionId, sessionId_value)`，调用 `onResult`，从 activeSubprocesses 删除该 sessionId
   - `'tool_use'`：调用 `onTool`，调用 `store.appendChatEvent(sessionId, 'agent.tool', { name, input })`
   - `'error'`：调用 `store.appendChatEvent(sessionId, 'session.error', { message: event.error?.message })`，调用 `onError`

   注意：`message_stop` 中 usage 字段在 claude --output-format stream-json 可能在单独的 `message` 事件中（或在 stop 事件本身的 message 字段）。实现时需累积一个 `lastUsage` 变量，从 `message` event（type='message'）取 `usage` 字段，在 `message_stop` 时使用。

7. **子进程退出处理**：
   - `child.on('close', ...)` 时若 activeSubprocesses 中还有该 sessionId（未被 message_stop 正常清除），说明崩溃，从 Map 删除即可（前端会因无 catchup 显示"回复丢失"）

8. **getCatchup(sessionId)**：返回 `activeSubprocesses.get(sessionId)?.accumulatedText`（undefined 表示无活跃 subprocess）

9. **kill(sessionId)**：`activeSubprocesses.get(sessionId)?.process.kill('SIGTERM')`，从 Map 删除

<objective>
Create the shared foundation for persistent Gateway behavior: provider commands become a shared whitelist, `~/.tether/config.json` becomes the local config source, and `node-pty` is upgraded before long-running Gateway ownership.
</objective>

<threat_model>
## Threat Model

| Threat | Severity | Mitigation |
|--------|----------|------------|
| Session creation API later accepts arbitrary command execution | high | Move provider command resolution to a shared whitelist and make plans depend on that whitelist. |
| Local config executes code or hides unsafe defaults | medium | Use JSON-only config helpers and default remote/API session creation to disabled. |
| Long-running Gateway leaks PTY file descriptors | medium | Upgrade `node-pty` to `1.2.0-beta.12` or newer before supervisor work closes. |
</threat_model>

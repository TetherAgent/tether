<objective>
Make provider commands use the persistent Gateway by default while preserving the current inline path through fallback and `--inline`.
</objective>

<threat_model>
## Threat Model

| Threat | Severity | Mitigation |
|--------|----------|------------|
| CLI forwarding sends arbitrary command fields to Gateway | high | Forward only provider, projectPath, cols, and rows; provider comes from shared whitelist. |
| Fallback hides a broken Gateway and confuses user | medium | Print Chinese messages for retry, fallback, and inline mode. |
| CLI accidentally attaches to stale registry entry | medium | Treat registry as hint only; verify `/api/status` before forwarding. |
</threat_model>

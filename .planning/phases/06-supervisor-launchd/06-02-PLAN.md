<objective>
Add the Gateway runtime surface that a persistent Gateway needs: a config-gated session creation endpoint, a status endpoint, and Relay connection state reporting.
</objective>

<threat_model>
## Threat Model

| Threat | Severity | Mitigation |
|--------|----------|------------|
| Remote clients create arbitrary processes | high | `POST /api/sessions` is disabled by default and accepts only shared whitelist provider names. |
| Command-shaped JSON slips through validation | high | Reject request payloads containing `command`, `args`, `argv`, `env`, `shell`, or `providerCommand` anywhere. |
| Invalid terminal sizes crash PTY resize/create | medium | Reuse `isValidTerminalSize`; use safe defaults when cols/rows are absent. |
| Relay status misleads user into thinking remote access is healthy | low | Expose explicit relay state from `relay-client.ts` and include it in status when configured. |
</threat_model>

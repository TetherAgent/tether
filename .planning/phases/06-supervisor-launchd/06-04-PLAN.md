<objective>
Implement the macOS background Gateway lifecycle: install, start, stop, restart, status, and uninstall through launchd with Chinese user-facing output and absolute plist entries.
</objective>

<threat_model>
## Threat Model

| Threat | Severity | Mitigation |
|--------|----------|------------|
| launchd command uses shell interpolation | high | All process calls use `spawn(cmd, args[])`; tests inspect helper code for no `shell:true`. |
| plist depends on cwd, `pnpm tether`, `$HOME`, or shell PATH | high | Generate plist with absolute node/source entry captured at install/start time. |
| `gateway start` creates a second supervisor path | medium | `start` ensures plist exists and starts via launchd only. |
| status output hides a failed background Gateway | medium | Combine plist state, registry/API reachability, PID, URL, and relay state. |
</threat_model>

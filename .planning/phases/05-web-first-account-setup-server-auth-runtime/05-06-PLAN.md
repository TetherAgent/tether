<objective>
Ship the user-facing Phase 5 Web auth surfaces: `/register`, `/login`, `/admin/register`, `/admin/login`, localStorage-backed auth state, and session-shell gating that keeps normal users on their own session surface without redesigning the terminal itself.
</objective>

<threat_model>
## Threat Model

| Threat | Severity | Mitigation |
|--------|----------|------------|
| Web reintroduces `/setup?token=...` semantics that conflict with current product decisions | medium | Use `/register` as the only required first-owner path and make any `/setup?token=...` route clearly out of the required path. |
| Browser auth state sends management tokens into terminal session flows | high | Keep separate storage keys and auth context paths for normal vs management login. |
| Auth guard breaks the existing session shell or bypasses it with stale localStorage state | medium | Validate auth state through `GET /api/auth/me` and redirect invalid state to `/login`. |
</threat_model>

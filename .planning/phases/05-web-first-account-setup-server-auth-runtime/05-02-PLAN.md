<objective>
Install the minimum Web foundation that Phase 5 auth pages require: Tailwind 4, shadcn, shared UI primitives, and an auth-aware session shell that can later host `/register`, `/login`, `/admin/register`, and `/admin/login` without redesigning the terminal surface.
</objective>

<threat_model>
## Threat Model

| Threat | Severity | Mitigation |
|--------|----------|------------|
| Web auth pages drift away from the approved component set and visual contract | medium | Only initialize the shadcn primitives listed in `05-UI-SPEC.md`: `button`, `input`, `form`, `card`, and `label`. |
| Vite proxy/auth shell changes break existing Gateway session UI | medium | Keep terminal/xterm shell intact; only layer auth routing and session guard around `apps/web/src/main.tsx`. |
| Tailwind/shadcn setup pulls in ad hoc registries or extra components that expand Phase 5 scope | low | Use official shadcn init and keep `components.json` / generated components minimal. |
</threat_model>

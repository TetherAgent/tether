<objective>
Upgrade Relay from shared-secret bootstrap auth to scoped token auth while preserving the non-executing routing boundary. After this plan, Relay should accept Gateway/client sockets only with valid token identity and should route frames only inside the authorized ownership envelope.
</objective>

<threat_model>
## Threat Model

| Threat | Severity | Mitigation |
|--------|----------|------------|
| Relay keeps accepting bootstrap secret as the silent default | high | Token auth becomes the default; legacy secret path works only behind `TETHER_RELAY_ALLOW_LEGACY_SECRET=1`. |
| Relay registers or routes sockets without enforcing account/workspace/gateway/session boundaries | high | Validate scope on connect and on frame routing; reject cross-account or cross-session traffic. |
| Relay accidentally becomes an execution authority while auth logic is being added | high | Do not add any provider command, process creation, or ownership persistence paths; keep the frame router boundary intact. |
</threat_model>

  <action>
    On registration, list, subscribe, input, resize, stop, and control frames, require the Relay connection's validated scope to match the target `accountId`, `workspaceId`, `gatewayId`, and `sessionId`.
    A client from account A must not list or control account B sessions even if it guesses IDs.
    A ticket for one session must not subscribe to a different session.
    Keep Relay as routing-only infrastructure: do not add command start, provider args, env forwarding, or session ownership persistence.
  </action>

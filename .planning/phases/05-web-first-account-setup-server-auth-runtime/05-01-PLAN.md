<objective>
Create the `apps/server` service boundary and the shared auth/schema contract that every later Phase 5 plan depends on. This plan must leave the repo with a bootable Egg TypeScript service skeleton, explicit environment/config keys, SQL bootstrap tables for account/workspace/auth state, and shared token/scope types that Gateway, Relay, and Web can all consume.
</objective>

<threat_model>
## Threat Model

| Threat | Severity | Mitigation |
|--------|----------|------------|
| `apps/server` starts with an empty JWT secret or wildcard CORS and silently accepts insecure defaults | high | `config.default.ts` must throw when `TETHER_SERVER_JWT_SECRET` is empty outside test and must restrict CORS origin to configured Web origin(s). |
| Schema omits ownership scope and later plans improvise auth boundaries | high | `001_init.sql` must define `accountId`, `workspaceId`, identity, token, and audit tables up front with concrete columns. |
| Shared types do not carry enough scope for ticket/gateway/relay authorization | high | `packages/protocol` and `packages/core` must add exact token payload fields for `accountId`, `workspaceId`, `gatewayId`, `sessionId`, `mode`, `tokenClass`, and `jti`. |
</threat_model>

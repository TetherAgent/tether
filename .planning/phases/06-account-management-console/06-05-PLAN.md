状态管理：
- `devices: AdminDevice[]` — 当前页设备列表
- `total: number` — 总数
- `page: number` — 当前页（默认 1）
- `loading: boolean` — 加载中
- `error: string | null` — 加载错误
- `revoking: string | null` — 正在吊销的设备 ID（Dialog 内 disable 按钮用）
- `revokeError: string | null` — 吊销操作错误
- `confirmDevice: AdminDevice | null` — 待确认吊销的设备（非 null 时打开 Dialog）

逻辑：
1. useEffect 调用 listDevices(token, page) 获取数据
2. "吊销"按钮点击 → `setConfirmDevice(device)` 打开 Dialog
3. Dialog "确认吊销" 点击：
   - `setRevoking(device.id)`（disable 按钮）
   - 调用 `revokeDevice(token, device.id)`
   - 成功：`setDevices(current => current.filter(d => d.id !== confirmDevice.id))`，关闭 Dialog（setConfirmDevice(null)），清空 revokeError
   - 失败：`setRevokeError('操作失败，请稍后重试。')`
   - finally：`setRevoking(null)`
4. Dialog "取消"：`setConfirmDevice(null)`，清空 revokeError

- h1: "设备" + 分隔

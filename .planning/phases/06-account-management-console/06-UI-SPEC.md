## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, badge padding, inline spacing |
| sm | 8px | Table cell padding (vertical), form field gap |
| md | 16px | Card padding, sidebar item vertical padding, form stack gap |
| lg | 24px | Section padding, page content padding (top/bottom) |
| xl | 32px | Layout gap between sidebar and content area |
| 2xl | 48px | Empty state vertical centering padding |
| 3xl | 64px | Page-level outer padding (unused in dashboard; reserved) |

Exceptions:
- Sidebar width: 220px (fixed, not from scale — layout constraint)
- Header height: 52px (fixed — balances icon + text at 14px)
- Table row height: 48px (touch-safe; within 8-point grid)
- Confirmation dialog max-width: 400px

| Role | CSS Variable | Resolved Hex (approx) | Usage |
|------|-------------|----------------------|-------|
| Dominant (60%) | `--background` | #101214 | Page background, content area fill |

- 离线 (offline): `text-muted-foreground` / `bg-muted/40`
- 已吊销 (revoked): `text-destructive` / `bg-destructive/10`

## Page Inventory

### 1. 概览 (Dashboard) — `/admin/dashboard`

Layout: 2-column stat card grid (at ≥768px), single column on mobile.

主视觉焦点：4 个统计数字（24px/600），是页面第一视觉落点；卡片标签为次要信息。

Stat cards (4 total):
- 已注册用户 (total users)
- 活跃设备 (active devices — not revoked)
- 注册 Gateway (registered gateways)
- 近 7 天审计事件 (audit events last 7 days)

Each stat card: `Card` component, display number (24px/600), label below (12px/600/muted).

### 2. 用户 (Users) — `/admin/users`

Table columns: 邮箱 / 注册时间 / 登录次数 / 登录失败次数 / 最后登录 / 活跃设备数

Pagination: 20 rows per page, shadcn `Pagination` component.

Row click: expand inline or navigate to user detail (planner decides — out of UI contract scope).

### 3. 设备 (Devices) — `/admin/devices`

Table columns: 设备名 / 类型 / 所属用户 / 在线状态 / 最后在线时间 / 操作

操作 column: "吊销" `Button` variant=destructive, size=sm.
Already-revoked rows: "已吊销" badge (red muted), no button.

Pagination: 20 rows per page.

Source: CONTEXT.md D-16, D-17, D-18, D-19.

### 4. Gateway — `/admin/gateways`

Table columns: Gateway ID / 最后认证时间 / 在线状态 / 操作

操作 column: "取消链接" `Button` variant=destructive, size=sm.
Already-unlinked rows: "已取消链接" badge, no button.

Online state: derived from server registration record (no real-time probe).
Source: CONTEXT.md D-20, D-21, D-22.

### 5. 审计 (Audit) — `/admin/audit`

Filter bar: 用户邮箱 (text input) + 操作类型 (select) + 时间范围 (date range picker or two date inputs).

Table columns: 时间 / 用户 / 设备 / Gateway / 操作类型 / 详情

Pagination: 50 rows per page (audit logs are high-volume).
Row detail: inline expand (not modal) — show full event JSON in `<pre>` with monospace 12px.

## Interaction Contracts

### Revoke Device Flow

1. Admin clicks "吊销" button on a device row.
2. `Dialog` opens with title "吊销设备" and body "确定吊销设备 {设备名}？此操作不可撤销。".
3. Dialog footer: "取消" (secondary) + "确认吊销" (destructive/primary).
4. On "确认吊销": disable both buttons, call `DELETE /admin/api/devices/:id`.
5. On success: close dialog, remove row from table in-place (optimistic removal, no page reload).
6. On API error: show inline error message inside dialog ("吊销失败，请稍后重试"), re-enable buttons.

Source: CONTEXT.md D-16, D-17, D-18, D-19.

### Unlink Gateway Flow

1. Admin clicks "取消链接" button on a gateway row.
2. `Dialog` opens with title "取消链接 Gateway" and body "确定取消链接 {Gateway ID}？该 Gateway 将无法再通过 Relay 发布会话。".
3. Dialog footer: "取消" (secondary) + "确认取消链接" (destructive/primary).
4. On "确认取消链接": disable both buttons, call `DELETE /admin/api/gateways/:id`.
5. On success: close dialog, remove row in-place.
6. On API error: inline error inside dialog, re-enable buttons.

Source: CONTEXT.md D-20, D-21, D-22.

### Table Loading State

- While awaiting API response: render `Skeleton` rows (same count as page size, height=48px).
- On error: replace table body with inline error message + "重试" button.
- Empty result: show empty state (see Copywriting Contract below).

### Auth Guard

- On mount, `AdminLayout` reads `localStorage.getItem('admin_token')`.
- If absent or expired (401 from any API call): clear token, redirect to `/admin/login`.
- Token key in localStorage: `admin_token` (consistent with Phase 5 pattern).
- Authorization header: `Authorization: Bearer <token>`.

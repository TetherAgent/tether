# Plan 06-06: 管理员注册页 + 登录后身份信息填充

**Phase:** 06 — Account Management Console
**Plan:** 06-06
**Wave:** 6

## Objective

修复 VERIFICATION.md 中识别的两个 BLOCKER。BLOCKER 1：添加 AdminRegisterPage 组件和 `/admin/register` 路由，让管理员可通过 UI 注册（后端 `POST /admin/api/admins` 端点已就绪）。BLOCKER 2：修复 `loginManagement` 在写入 localStorage 前不解析 JWT payload 的问题，使 AdminLayout header 能正确显示已登录管理员的 email。

## Gap Context

**Addresses:** BLOCKER 1 (SC-2) — 无注册页面；BLOCKER 2 (SC-3) — identity 字段从未填充

**Root Cause (BLOCKER 1):** `main.tsx` 没有 `/admin/register` 路由，`apps/admin-web/src/pages/` 中没有注册页面组件。服务端 `POST /admin/api/admins` 端点已存在，但前端 UI 缺失。

**Root Cause (BLOCKER 2):** `loginManagement` 收到 `accessToken` 后直接存储 `{ accessToken, refreshToken }` 而不解析 JWT payload。`ManagementIdentity` 类型已定义但 `identity` 字段从未被赋值。

**Not addressed:** BLOCKER 3 (SC-4) 角色权限差异化已按 CONTEXT.md D-13/D-14 推迟——v0.3 单一管理访问级别，登录即全权，无需 UI 层角色判断。

export async function createAdmin(token: string, data: {
  email: string;
  password: string;

**Step 1: 在文件顶部（type 定义之前）添加 JWT 解析工具函数**

```typescript
function decodeJwtPayload(token: string): Record<string, unknown> | null {
  try {
    const parts = token.split('.');
    if (parts.length !== 3) return null;
    const payload = parts[1];
    // atob 解码 base64url（将 - 换 +，_ 换 /）
    const base64 = payload.replace(/-/g, '+').replace(/_/g, '/');
    const json = atob(base64);
    return JSON.parse(json) as Record<string, unknown>;
  } catch {
    return null;
  }
}
```

**Step 2: 修改 `loginManagement` 回调函数**

将当前的：
```typescript
const record: AuthStorageRecord<ManagementIdentity> = {
  accessToken: body.accessToken,
  refreshToken: body.refreshToken
};
```

替换为（在 `writeStorage` 调用之前）：
```typescript
const jwtPayload = decodeJwtPayload(body.accessToken);
const identity: ManagementIdentity | undefined = jwtPayload && typeof jwtPayload.adminUserId === 'string'
  ? {
      adminUserId: jwtPayload.adminUserId as string,
      email: typeof jwtPayload.email === 'string' ? jwtPayload.email : '',
      accountId: typeof jwtPayload.accountId === 'string' ? jwtPayload.accountId : '',
      workspaceId: typeof jwtPayload.workspaceId === 'string' ? jwtPayload.workspaceId : '',
      tokenClass: 'management_access',
      expiresAt: typeof jwtPayload.exp === 'number' ? jwtPayload.exp : 0,
      jti: typeof jwtPayload.jti === 'string' ? jwtPayload.jti : ''
    }
  : undefined;
const record: AuthStorageRecord<ManagementIdentity> = {
  accessToken: body.accessToken,
  refreshToken: body.refreshToken,
  identity
};
```

**约束：**
- 不修改 `ManagementIdentity` 类型定义——它已经包含所有必需字段
- 不修改 `readStorage` 函数——已登录用户从 localStorage 读取时 identity 已保存在存储中，下次刷新会正常恢复
- 不修改 `writeStorage` 函数
- `decodeJwtPayload` 中使用 try/catch 保证解析失败时不影响登录流程（identity 为 undefined 而非抛出异常）

**验证 AdminLayout.tsx 消费方式（无需修改）：**
`AdminLayout.tsx` 第 23 行使用 `managementAuth.identity?.adminUserId ?? ''`。填充 identity 后，若 JWT payload 包含 `email` 字段，可将该行改为 `managementAuth.identity?.email ?? managementAuth.identity?.adminUserId ?? ''` 以显示 email 而非 UUID。如果 JWT payload 确实包含 email，则同步更新 AdminLayout.tsx 的显示逻辑。

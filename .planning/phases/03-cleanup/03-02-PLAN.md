<objective>
Delete the Gateway tmux subprocess wrapper and replace runtime tmux branches with PTY-only behavior plus explicit legacy unsupported responses for historical rows.
</objective>

<threat_model>
## Threat Model

| Threat | Severity | Mitigation |
|--------|----------|------------|
| Gateway continues executing tmux commands after Phase 3 | high | Delete `apps/gateway/src/tmux.ts`, remove exports, and static-scan for executable helper names. |
| Historical rows trigger accidental process probing or killing | high | Return HTTP 409 for legacy rows and never call `tmux has-session`, `capture-pane`, `send-keys`, or `kill-session`. |
| Snapshot/send/stop behavior regresses for PTY sessions | medium | Add HTTP tests for PTY snapshot/send/stop and legacy 409 paths. |
</threat_model>

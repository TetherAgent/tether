<objective>
Remove the user-visible `--transport` option and all CLI tmux operations so CLI commands only create, attach, send to, list, and stop PTY-backed sessions.
</objective>

<threat_model>
## Threat Model

| Threat | Severity | Mitigation |
|--------|----------|------------|
| Users can still create tmux sessions through CLI flags | high | Remove `--transport` from provider commands and `tether run`; Commander should reject the unknown option. |
| CLI directly reads historical rows and executes tmux attach/send | high | Replace tmux branches with a Chinese legacy unsupported error. |
| `stop --all` attempts to stop historical lost rows | medium | Filter running session IDs by active PTY transport as well as status. |
</threat_model>

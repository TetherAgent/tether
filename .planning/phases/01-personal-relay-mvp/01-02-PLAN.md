<objective>
Create `apps/relay` as an independent Node/TypeScript relay service. It authenticates one owner Gateway and remote Web clients with `TETHER_RELAY_SECRET`, keeps only in-memory routing state, and forwards protocol frames without serving `apps/web` static assets.
</objective>

<threat_model>
## Threat Model

| Threat | Severity | Mitigation |
|--------|----------|------------|
| Public relay accepts unauthenticated Gateway or client sockets | high | Require `TETHER_RELAY_SECRET`; close unauthenticated sockets with code `1008`. |
| Relay stores terminal plaintext | high | Keep no transcript/event persistence in `apps/relay`; only forward frames in memory. |
| Relay becomes remote execution surface | high | Reject unknown frames and any payload containing `command`, `args`, `argv`, `env`, or `providerCommand`. |
| Nginx/static Web deployment is confused with relay service | medium | Do not serve `apps/web/dist` from `apps/relay`; expose only relay WS endpoints and health. |
</threat_model>

    Support WebSocket path `/gateway` for exactly one authenticated Gateway connection and `/client` for authenticated browser clients.
    Require `options.secret` for auth frames; close unauthenticated or bad-secret sockets with close code `1008`.
    Keep in-memory maps for current gateway socket, client sockets, client subscriptions by `sessionId`, and latest sessions received from `gateway.sessions`.
    Forward `client.list`, `client.subscribe`, `client.input`, `client.resize`, and `client.detach` from authenticated clients to the authenticated Gateway.

    Implement `apps/relay/src/main.ts` reading `TETHER_RELAY_SECRET`, `TETHER_RELAY_HOST` defaulting to `127.0.0.1`, and `TETHER_RELAY_PORT` defaulting to `4889`; print `Tether Relay: http://HOST:PORT`; wait for SIGINT/SIGTERM before close.

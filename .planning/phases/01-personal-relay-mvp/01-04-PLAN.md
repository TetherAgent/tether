<objective>
Reuse the existing `apps/web` UI through a relay connection mode, document nginx-based deployment, and verify the full Gateway -> Relay -> remote Web path locally and against the user's Node server.
</objective>

<threat_model>
## Threat Model

| Threat | Severity | Mitigation |
|--------|----------|------------|
| Browser relay secret leaks through logs or URLs | high | Prompt/store secret in localStorage; do not require secret in URL query params. |
| Relay mode duplicates UI and diverges from direct mode | medium | Introduce a connection adapter while preserving existing SessionList/PtySessionView concepts. |
| Nginx deployment misroutes WS paths | medium | Document exact Web static and Relay WS/API responsibilities. |
| End-to-end validation misses direct Gateway regression | high | Verify both relay mode and existing direct Gateway mode after implementation. |
</threat_model>

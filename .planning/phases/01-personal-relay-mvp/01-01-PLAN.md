<objective>
Replace the placeholder relay frame union with a formal TypeScript relay protocol contract shared by Gateway, Relay, and Web. This plan creates the source of truth for all Phase 1 frame shapes before runtime code is implemented.
</objective>

<threat_model>
## Threat Model

| Threat | Severity | Mitigation |
|--------|----------|------------|
| Relay protocol accepts command/args/env and becomes remote process execution | high | Do not include provider command, argv, env, or project-path spawn frames in any exported client-to-relay type. |
| Gateway and client frames get mixed, allowing browser clients to impersonate Gateway registration | high | Export separate directional unions for Gateway-to-Relay, Relay-to-Gateway, Client-to-Relay, and Relay-to-Client frames. |
| Future auth work cannot reuse the protocol because secret auth is hard-coded everywhere | medium | Model auth as explicit frames with `secret` in MVP, keeping frame names extensible for device tokens later. |
</threat_model>

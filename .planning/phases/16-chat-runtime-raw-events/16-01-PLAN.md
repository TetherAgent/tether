创建文件 `apps/server/sql/005-chat-runtime-events.sql`，包含以下两部分：

**Part 1 — 新建表（幂等）：**
```sql
CREATE TABLE IF NOT EXISTS gateway_runtime_chats_events (
  id          BIGINT UNSIGNED AUTO_INCREMENT PRIMARY KEY,
  session_id  VARCHAR(128)    NOT NULL,
  event_id    INT             NOT NULL,
  event_type  VARCHAR(64)     NOT NULL,
  raw_json    MEDIUMTEXT      NOT NULL,
  created_at  TIMESTAMP       NOT NULL DEFAULT CURRENT_TIMESTAMP,
  updated_at  TIMESTAMP       NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  UNIQUE KEY  uk_session_event (session_id, event_id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4
```

**Part 2 — 条件 ALTER 加列（INFORMATION_SCHEMA 守卫，防止重复执行报错）：**
db.ts 的 splitSqlStatements 以分号分割，不支持 SET/PREPARE/EXECUTE 多语句块，因此 ALTER 必须拆开用 INFORMATION_SCHEMA 条件检查写成单条 SQL 或用 IF NOT EXISTS 语法。
MySQL 5.7+ 不支持 `ADD COLUMN IF NOT EXISTS`，但 MySQL 8.0 也不支持（标准不含此语法）。
正确方案：用 `INFORMATION_SCHEMA` SET + PREPARE + EXECUTE 模式，但 db.ts 禁用了多语句。
替代方案：用 `ALTER TABLE ... ADD COLUMN IF NOT EXISTS` 语法，MySQL 8.0 实际支持（注意 db.ts 已配置 multipleStatements: true）。

查看 db.ts config，MySQL client 开了 multipleStatements: true。因此可以用 INFORMATION_SCHEMA + SET/PREPARE/EXECUTE 多语句块，只需确保整块作为单个 SQL entry 被执行。

实际正确做法：在文件中用单个完整的多语句块，db.ts 会以分号分割；SET/PREPARE/EXECUTE 每行本身已是独立语句。db.ts 的 splitSqlStatements 按分号 split，DEALLOCATE 行后也有分号，所以每条都会单独执行。

**最终正确格式（每条语句之间用分号，db.ts 会逐条执行）：**

```sql
SET @p16_col_exists = (SELECT COUNT(*) FROM INFORMATION_SCHEMA.COLUMNS WHERE TABLE_SCHEMA = DATABASE() AND TABLE_NAME = 'gateway_chat_messages' AND COLUMN_NAME = 'raw_json');
SET @p16_ddl = IF(@p16_col_exists = 0, 'ALTER TABLE gateway_chat_messages ADD COLUMN raw_json MEDIUMTEXT NULL', 'SELECT 1 /* raw_json already exists */');
PREPARE p16_stmt FROM @p16_ddl;
EXECUTE p16_stmt;
DEALLOCATE PREPARE p16_stmt
```

注意：这五条 SQL 之间以分号分割，db.ts 会逐条执行，每条都可独立执行，幂等安全。
变量名加 `p16_` 前缀防止与其他 migration 的 SET 变量冲突。

完整文件内容：将 Part 1 (CREATE TABLE IF NOT EXISTS) 和 Part 2 (SET/PREPARE/EXECUTE) 写入同一文件，用分号分隔各语句。

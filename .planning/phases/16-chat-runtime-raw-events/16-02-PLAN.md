找到 `onDelta: (event: { clientId: string; sessionId: string; text: string }) => void;`
改为：

**2. 在 ActiveSubprocess 结构中添加 nextDeltaId 字段**：
找到 ActiveSubprocess 类型/接口的定义，在其中添加：
`nextDeltaId: number;`

**3. 在 activeSubprocesses.set() 时初始化 nextDeltaId**：
找到所有 `activeSubprocesses.set(sessionId, {...})` 的调用，在对象中添加：
`nextDeltaId: 1,`

**4. 修改 createEmitter 的 delta 回调**（约 lines 370-373）：
找到：
```typescript
delta: (text) => {
  active.accumulatedText += text;
  this.options.onDelta({ clientId: active.clientId, sessionId, text });
},
```
改为：
```typescript
delta: (text) => {
  active.accumulatedText += text;
  const deltaEventId = active.nextDeltaId++;
  this.options.onDelta({ clientId: active.clientId, sessionId, text, deltaEventId });
},
```
注意：`active.nextDeltaId++` 后置自增，先取值（1）再加到2，即第一条 delta id=1。

**5. 修改 finishResult 中的 resultEvent 构建**（约 lines 405-409）：
找到：
```typescript
const resultEvent = createChatEvent(sessionId, 'agent.result', {
  text,
  usage,
  ...(stopReason ? { stop_reason: stopReason } : {})
});
```
改为：
```typescript
const lastDeltaEventId = active.nextDeltaId > 1 ? active.nextDeltaId - 1 : 0;
const resultEvent = createChatEvent(sessionId, 'agent.result', {
  text,
  usage,
  lastDeltaEventId,
  ...(stopReason ? { stop_reason: stopReason } : {})
});
```
解释：`nextDeltaId` 初始为1，未产生任何 delta 时仍为1，此时 lastDeltaEventId=0（语义为"无 delta"）；产生过 delta 时 nextDeltaId > 1，lastDeltaEventId = nextDeltaId - 1 = 最后一条 delta id。

**重要：** 修改 onDelta 签名后，调用点 daemon.ts 或 relay-client.ts 中的 onDelta 实现也需同步更新（添加 deltaEventId 字段但可先忽略），否则 TypeScript 报错。搜索 `onDelta:` 的所有实现，添加 `deltaEventId` 参数（即使不使用也可用 `_` 前缀接收）。

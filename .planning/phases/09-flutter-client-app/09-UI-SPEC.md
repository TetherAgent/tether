# Phase 9 — UI Design Contract: Flutter Client App

> Visual and interaction contract for the phone-first Flutter agent console.
> Generated by gsd-ui-researcher. Verified by gsd-ui-checker.

## Design System

| Property         | Value                                                                 |
|------------------|-----------------------------------------------------------------------|
| Tool             | none — Flutter Material 3 (dark theme, manual tokens)                |
| Preset           | not applicable — Flutter, not shadcn/web                             |
| Component library| Flutter Material 3 (ThemeData.dark())                                 |
| Icon library     | Material Icons (bundled with Flutter)                                 |
| Font (body)      | System default sans-serif (Inter via GoogleFonts if available on platform) |
| Font (terminal)  | System monospace (Cascadia Code / Menlo / Consolas — platform fallback chain) |

**shadcn Gate Result:** Not applicable — this is Flutter, not React/Next.js/Vite. No shadcn initialization performed.

**Web Parity Note:** Color tokens are inherited from `apps/web/src/styles.css` (the established Web design language for Tether). Flutter replicates the same dark palette in Dart `ThemeData` constants so both surfaces feel unified.

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage                                                        |
|-------|-------|--------------------------------------------------------------|
| xs    | 4px   | Icon gaps, chip inner padding, badge padding                 |
| sm    | 8px   | Card internal row gaps, toolbar button gaps, status badge    |
| md    | 16px  | Default screen horizontal padding, card internal padding     |
| lg    | 24px  | Section heading bottom margin, empty state vertical padding  |
| xl    | 32px  | Auth screen vertical centering padding                       |
| 2xl   | 48px  | Auth card max-width margin on narrow phones                  |
| 3xl   | 64px  | (reserved — not used in phase 9 UI)                          |

**Exceptions (touch targets and component heights only):**

- Minimum touch target for all tappable elements: **48px** (iOS/Android HIG requirement; keyboard toolbar buttons are 48px tall)
- Keyboard toolbar height: **48px** exactly (matches iOS software keyboard toolbar standard)

## Component Fixed Dimensions

These are fixed component sizes determined by content and platform convention. They are not subject to the 8-point spacing grid and must not be moved to the spacing scale.

| Component                        | Dimension | Rationale                                                        |
|----------------------------------|-----------|------------------------------------------------------------------|
| Terminal widget inset padding    | 8px       | Closest compliant value to the Web's 6px terminal-host padding; changed from 6px to maintain 8-point grid compliance while preserving the intent of minimal terminal inset |
| Session card swipe-reveal width  | 80px      | Sufficient width to display "停止" label at 12sp; fixed by content, not grid |

## Typography

| Role     | Size  | Weight          | Line Height | Usage                                                                                          |
|----------|-------|-----------------|-------------|------------------------------------------------------------------------------------------------|
| Display  | 20sp  | SemiBold (600)  | 1.2         | Screen titles (app bar "Tether"), auth headings                                                |
| Body     | 15sp  | Regular (400)   | 1.5         | Session card title, form field text, body copy                                                 |
| Label    | 12sp  | Regular (400)   | 1.4         | Session card meta (provider · status · time), status bar text, section headings (uppercase), toolbar button labels, overlay badges (client-strip), reconnect countdown |
| Terminal | 13sp  | Regular (400)   | 1.25        | xterm terminal widget — monospace font, matches Web                                            |

**Weight rule:** Only two weights in use — Regular (400) for all body/label copy; SemiBold (600) for screen titles, session card titles, and primary CTA buttons.

**Visual distinction for overlay badges and reconnect countdown:** These elements use 12sp Label with `tetherMuted` color (rather than `tetherForeground`) to create visual hierarchy without a separate font size.

**Font size note for accessibility:** Terminal font starts at 13sp; pinch-to-zoom scales this value. Min clamp: 10sp. Max clamp: 24sp.

| Foreground      | `#e8ecef` | `tetherForeground`         | Primary text, session card title, input text                           |
| Muted           | `#9aa4af` | `tetherMuted`              | Session card meta, status bar text, section headings, placeholder text, overlay badges, reconnect countdown |
| Destructive     | `#e05252` | `tetherDestructive`        | Swipe-to-reveal stop action background, error messages, token expired toast |

**60/30/10 breakdown:**
- 60% dominant: terminal area, list background, auth center background — the app is terminal-first, terminal gets the darkest surface
- 30% secondary: all chrome surfaces (app bar, session cards, keyboard toolbar, input fields)
- 10% accent: reserved exclusively for — primary CTA button fills, active mode-toggle ring, keyboard toolbar Ctrl-held indicator, session card hover/focus border, and the cursor in xterm

## Screens and Interaction Contract

### Screen 1: Login Screen

**Route:** Initial screen when no valid token present.

**Layout:**
- Centered card (max-width 360dp, horizontal padding 16dp)
- Logo/wordmark "Tether" at 20sp SemiBold, centered
- Subtitle: "Agent 控制台" at 12sp muted
- Email field + password field stacked at 16dp gap
- Primary CTA button full-width below fields, 48dp height
- "还没有账号？注册" footer link at 15sp muted

**Interaction states:**
- Loading: button shows `CircularProgressIndicator` (white, 16dp), disabled; fields disabled
- Error: red banner below button with `tetherDestructive` text, 15sp, line-height 1.5
- Success: push to Session List screen, token stored via `flutter_secure_storage`

### Screen 2: Register Screen

**Route:** Accessed from login footer link.

**Layout:** Same card structure as Login.
- Display name field + Email field + Password field + Confirm password field
- Primary CTA: "注册"
- "已有账号？登录" footer link

**Interaction states:** Same as Login. Validation errors shown inline below field at 12sp `tetherDestructive`.

### Screen 3: Session List Screen

**Layout:**
- AppBar: "Tether" title (20sp SemiBold) + settings icon button (right, 48px touch target)
- Body: scrollable list with two sections

**主视觉焦点：** 活跃 session 区域的第一张卡片；AppBar 标题 "Tether" 作为品牌锚点。

**Section: 活跃** (active sessions)
- Section header: "活跃" at 12sp uppercase muted
- Session cards (see Card spec below)
- If empty and no history: empty state (see Copywriting)

**Section: 历史** (collapsible)
- `ExpansionTile` or `details`-equivalent: "历史" label at 12sp uppercase muted with arrow
- Up to 8 history session cards (same card layout, status label in muted color)

**Session Card spec:**
- Background: `tetherSurface` (#171a1f)
- Border: 1px `tetherBorder` (#2b3137), radius 8px
- Internal padding: 12dp
- Row 1: provider name + status badge (12sp label)
- Row 2: `projectPath` at 12sp muted, overflow ellipsis
- Row 3: `lastActiveAt` timestamp at 12sp muted
- Status badge colors: running → `tetherAccent` dot; stopped/completed → `tetherMuted` dot; failed/lost → `tetherDestructive` dot
- **Swipe-to-reveal (active sessions only):** Slide left to reveal "停止" action (red background `tetherDestructive`, white 12sp label, 80dp wide — see Component Fixed Dimensions). Applies only to running sessions.
- Tap card → navigate to Terminal Screen for that session

**Connection state indicator:**
- Small status pill at top-right of AppBar (12sp, muted background)
- States: "正在连接…" / "Relay · {clientId:8}" / "已断开" / "连接错误"

### Screen 4: Terminal Screen

**Layout (portrait):**
- AppBar: back arrow + session title/provider (truncated) + mode toggle + status pill
- Body: xterm terminal widget, fills available height, no padding except 8px inset (see Component Fixed Dimensions)
- Keyboard Toolbar (shown above software keyboard when keyboard is open): 48dp height, `tetherSurface` background, 3 buttons equally spaced: **Ctrl** / **Esc** / **Tab**

**Layout (landscape):**
- Same as portrait but terminal widget has more rows
- Keyboard toolbar still 48dp, attaches to keyboard

**AppBar buttons:**
- Back button (←): 48px touch target, navigates back to Session List
- Mode toggle: pill/chip showing "控制" or "观察", tapping toggles. "控制" = `tetherAccent` text/ring; "观察" = muted
- Status pill: "Relay · abc12345" or reconnect countdown (12sp muted)

**Keyboard Toolbar button spec:**
- Width: 1/3 of toolbar width each
- Height: 48dp
- Label: "Ctrl" / "Esc" / "Tab" at 12sp Regular
- Background normal: `tetherSurface`
- Background active (Ctrl held): `tetherAccent` at 20% opacity, border `tetherAccent`
- No icon — label only

**Ctrl modifier behavior:** Tap Ctrl → enters "Ctrl-held" state (visual highlight); next character key sends `ctrl + char` byte. Held state resets after one keypress or after 3 seconds with no input.

**Pinch-to-zoom:** Two-finger pinch inside terminal adjusts `fontSize` on xterm widget. Min 10sp, max 24sp. No UI control shown — gesture only.

**Orientation:** Free rotation. On rotation, terminal fires resize event with new cols/rows.

**Overlay badge (client-strip):**
- Positioned bottom-center when keyboard closed, or above keyboard toolbar when open
- Shows: mode label ("控制" / "观察") + connection mode pill ("Relay" / "LAN")
- 12sp Label, `tetherMuted` text, `tetherBackground` bg at 86% opacity, 1px border `tetherBorder`, pill border-radius

### Screen 5: Settings Screen (partial — connection mode + LAN address only)

**Accessed via:** Settings icon in Session List AppBar.

**Layout:**
- Standard `ListView` with `ListTile` groups
- Section "连接": mode selector (Relay / LAN 直连)
- If LAN mode: text field for Gateway base URL (e.g., `http://192.168.1.x:4789`)
- Apply button or auto-save on field blur

**Deferred fields (visible but disabled or absent):**
- Server URL: not shown in this phase (D-08 deferred)
- Arrow keys toolbar row: not shown (deferred)

## Interaction States (all screens)

| State          | Visual treatment                                                                 |
|----------------|----------------------------------------------------------------------------------|
| Loading        | `CircularProgressIndicator` (16dp, `tetherAccent`) inline with disabled controls |
| Error          | Red text `tetherDestructive` 15sp line-height 1.5, no modal; shown inline below trigger |
| Empty          | Centered column: heading 20sp + body 15sp muted, 24dp gap                        |
| Reconnecting   | Status pill pulses opacity (0.4 → 1.0 at 1s cycle), countdown label updates     |
| Disconnected   | Status pill text: "已断开" in muted; terminal surface stays visible (last frame)  |
| Observe mode   | Terminal accepts no keyboard/toolbar input; toolbar buttons visually dimmed (0.4 opacity) |

## Accessibility

- All buttons: minimum 48dp touch target
- Color is never the sole differentiator for session status — status labels accompany dots
- Terminal text: dynamic font size (pinch zoom) minimum 10sp
- VoiceOver/TalkBack labels on AppBar icon buttons ("设置", "返回", "切换模式")
- Keyboard toolbar buttons have semantic labels ("Ctrl 键", "Esc 键", "Tab 键")

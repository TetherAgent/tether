1. 类型扩展

2. 乐观写入

改 sendChatText,删掉那 3 次轮询:

diff 
  setIsSending(true);
  try {
    const localId = `pending:${crypto.randomUUID()}`;
    setChatMessages((p) => [...p, { id: localId, role: 'user',
      content: nextValue, tools: [], status: 'pending' }]);
    sendFrame(nextValue);
    await wait(CHAT_ENTER_DELAY_MS);
    sendFrame('\r');
    setChatMessages((p) => p.map((m) => m.id === localId
      ? { ...m, status: 'sent' } : m));
    window.setTimeout(() => void refreshConversation(), 250);
    window.setTimeout(() => void refreshConversation(), 1250);
    window.setTimeout(() => void refreshConversation(), 3000);
    setInputText('');
  } finally { setIsSending(false); }

这样从用户点击到看见消息只有 1 帧延迟。

关键点

  • 不要按内容比对去重 — WebSocket 顺序保证 + FIFO 匹配 pending 队列
  • 5 秒超时 → 标 failed + 重试按钮
  • 删 3 次轮询时小心 echo 延迟,改完一起测

状态对照表

状态触发视觉

注意 refreshConversation 删掉之后,如果 echo 比预期晚,本地永远卡 sent — 必须把 5s 超时机制和删除轮询同时落地。
